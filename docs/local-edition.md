# Local Edition

本地版是 `GalaxyOPC` 面向非技术用户传播和落地的重点形态。

GitHub 开源仓库适合开发者查看源码、参与协作和建立技术信任；但真正的圈内传播，不能要求用户先学会 Git、Node.js、数据库和命令行。因此本地版的目标是：下载安装后，通过几个简洁配置界面，就能马上开始使用。

## 产品原则

- 少配置：默认可用，复杂项后置。
- 先体验：提供可试用的星环算力通道。
- 可自托管：用户仍可切换到自己的模型、数据库和服务。
- 真入口：飞书、企业微信、钉钉要成为真实工作入口。
- 空间化：城市和物联网页面要独立表达物理世界。

## 默认体验路径

1. 用户安装 Windows / macOS 本地版。
2. 打开应用后进入欢迎页。
3. 选择“立即体验”或“高级配置”。
4. 默认使用本地 SQLite 和星环试用模型通道。
5. 创建第一个公司或工作空间。
6. 进入 AI 对话。
7. 后续再配置飞书、企业微信、钉钉、邮箱、搜索和私有模型。

## 配置向导建议

### 1. 模型配置

提供两个入口：

- 星环试用通道：默认可用，适合先体验。
- 自配 API Key：适合长期使用和私有部署。

自配 API Key 页面应预设常见模型官网链接：

- 通义千问 DashScope: `https://dashscope.console.aliyun.com/`
- OpenAI Platform: `https://platform.openai.com/api-keys`
- DeepSeek Platform: `https://platform.deepseek.com/`
- 智谱 AI 开放平台: `https://open.bigmodel.cn/`
- Moonshot AI: `https://platform.moonshot.cn/`
- OpenRouter: `https://openrouter.ai/keys`

### 2. 搜索配置

默认可以先关闭，用户需要联网情报能力时再配置。

建议提供：

- 星环默认搜索通道
- 自配搜索 API
- 不启用搜索

### 3. 协作通道

优先顺序：

- 飞书
- 企业微信
- 钉钉

每个通道都应该包含：

- 官网配置入口
- App ID / Secret / Agent ID 等字段说明
- 权限清单
- 连接测试
- 示例指令

### 4. 本地数据

本地版默认使用 SQLite。

用户需要明确看到：

- 数据存放在哪里
- 如何备份
- 如何迁移
- 如何清空

## 页面结构建议

### AI 对话入口

负责自然语言调度，是用户每天最常打开的入口。

### 管理后台

负责软件系统管理，包括：

- 账号
- 公司
- 项目
- 订单
- 配置
- 套餐
- 日志

### 城市空间

独立横向页面，负责表达城市节点和 OPC 中心：

- 城市地图
- OPC 中心
- 园区/节点
- 资源分布
- 招募与申请

### 物联网空间

独立横向页面，负责表达物理空间和设备：

- 楼层
- 房间
- 设备
- 3D 布局
- 设备清单
- 空间方案导出

## MVP 清单

- Windows 安装包
- macOS 安装包
- 本地 SQLite 默认启动
- 首次配置向导
- 星环试用模型通道
- 自配模型 API Key 页面
- 飞书连接产品化
- 企业微信和钉钉通道骨架
- 城市管理横向页面
- 物联网管理横向页面
